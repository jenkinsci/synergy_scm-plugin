Name des Projektes in der Form <project name><delimiter><version>.

Wenn das Projekt ein development oder testing Projekt ist, das Projekt also im Status working oder prep ist, wird der workarea path auf Hudsons workspace gesetzt und das Projekt aktualisiert.
Wenn das Projekt Subprojekte enthält, werden deren Workareas relativ zum Elternprojekt gesetzt.

Wenn das Projekt eine Baseline bildet (das Projekt ist dann static), wird ein Snapshot in Hudsons workspace kopiert.

Es ist möglich Buildparameter zu injizieren. Beispielsweise wird bei einem Projektnamen project~${version} der entsprechende Wert des Buildparameters version für ${version} gesetzt.
